Код нужно оформлять так, чтобы он был понятным и простым в поддержке.

Для этого используют специальные наборы правил — стандарты. 

В PHP самыми распространенными являются стандарты PSR — PHP Standards Recommendations или стандартные рекомендации PHP от PHP-FIG. 

https://www.php-fig.org/psr/

PSR-1
Классы именуются в верблюдем стиле OneWorldSteel
с 1-й заглавной буквой

Константы - в верхнем регистре с использованием символа подчёркивания для разделения слов
const ONE_WORLD_STEEL = 1;

Методы (функции) в camelCase стиле
oneWorldSteels(...)

UTF-8 без BOM-байта

Линтеры - проверяют код на соответствие стандартам
PHP_CodeSniffer

Строкой мы считаем все, что находится внутри кавычек, даже если это пробел, один символ или вообще отсутствие символов.

Для экранирования используется обратный слеш \:

print_r("Dragon's mother said \"No\"");

Символ перевода строки в тексте - \n
print_r("Dragon\n");

print_r("Dragon" . PHP_EOL);

\r\n	- для Windows

Конкатенация - соединение строк

print_r('Dragon' . 'stone');
